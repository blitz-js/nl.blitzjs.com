Blitz gebruikt de `App` component in `app/pages/_app.ts` om pagina's te
initialiseren. Je kan hem wijzigen om geweldige dingen te doen zoals:

- Het behouden van de layout wanneer een pagina verandert
- Het behouden van de staat bij het navigeren door pagina's
- Aangepaste foutafhandeling met `componentDidCatch`
- Het injecteren van extra gegevens in pagina's
- [Het toevoegen van globale CSS](./css)

Het standaard `_app.tsx` bestand ziet er als volgt uit:

```tsx
// app/pages/_app.tsx
export default function App({ Component, pageProps }) {
  return <Component {...pageProps} />
}
```

De `Component` prop is de actieve pagina, dus wanneer je tussen routes
navigeert, zal `Component` veranderen naar de nieuwe pagina. Daarom zullen
alle props die je naar `Component` stuurt door de pagina worden ontvangen.

- Adding a custom `getInitialProps` in your `App` will disable
  [Automatic Static Optimization](./pages#automatic-static-optimization)

- When you add `getInitialProps` in your custom app, you must
  `import App from "blitz"`, call `App.getInitialProps(appContext)` inside
  `getInitialProps` and merge the returned object into the return value.
- `App` currently does not support page data fetching methods like
  [`getStaticProps`](./get-static-props) or
