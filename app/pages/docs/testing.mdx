## Jest {#jest}

[Jest](https://jestjs.io/) is een aangenaam testing framework dat
eenvoudig te gebruiken is. Om testen te draaien in je applicatie kan je
`jest` of `jest --watch` uitvoeren in je terminal.

Om een nieuwe test te maken, maak je een bestand aan met de naam
`something.test.ts` (of `.js`, `.tsx`, etc.). Hier is een voorbeeld:

```ts
import something from "./somewhere"

it("does something", () => {
  const result = something()
  expect(result).toBe(true)
})
```

> Voor meer details over het gebruik van Jest,
> [lees de documentatie](https://jestjs.io/docs/en/using-matchers)

### Jest Config {#jest-config}

Blitz biedt een Jest preset die veel complexe zaken voor jou configureert,
inclusief Typescript ondersteuning.

Als je
[de Jest configuratie moet aanpassen](https://jestjs.io/docs/en/configuration),
kan je dit doen in `jest.config.js`.

```ts
// jest.config.js
module.exports = {
  preset: "blitz",
}
```

### Client- vs. serveromgevingen {#client-vs-server-environments}

De volgende testbestanden zullen in de Jest serveromgeving uitgevoerd
worden:

- `*.test.*` bestanden in een `api`, `queries`, of `mutations` map
- `*.server.test.*` bestanden

Alle andere bestanden zullen in de Jest clientomgeving uitgevoerd worden
(met jsdom).

## @testing-library {#testing-library}

pakketten die je helpt om UI componenten te testen op een
gebruikersgerichte manier. Het is een beter alternatief voor Enzyme, als
je dit in het verleden al gebruikt hebt.
