In Blitz is een **pagina** een
[React component](https://reactjs.org/docs/components-and-props.html) die
geÃ«xporteerd wordt vanuit een `.js`, `.jsx`, `.ts` of `.tsx` bestand in
een `pages` map. Elke pagina wordt geassocieerd met een route op basis van
zijn bestandspad.

Alle volgende pagina's zijn geldig:

- `app/pages/about.tsx`
- `app/projects/pages/projects/index.tsx`
- `app/tasks/pages/projects/[projectId]/tasks/[taskId].tsx`

**Voorbeeld**: Als je `app/pages/about.js` maakt die een React component
exporteert zoals hieronder, zal deze beschikbaar zijn op `/about`.

```jsx
function About() {
  return <div>Over ons</div>
}

export default About
```

### Pagina's met dynamische routes {#pages-with-dynamic-routes}

Blitz ondersteunt pagina's met dynamische routes. Bijvoorbeeld, als je een
bestand met de naam `app/pages/posts/[id].js` maakt, dan zal het
beschikbaar zijn op `posts/1`, `posts/2`, etc.

> Voor meer informatie over routing, lees de
> [routing documentatie](./routing).

### Automatische statische optimalisatie {#automatic-static-optimization}

Standaard pre-rendert Blitz de statische HTML voor elke pagina, tenzij je
expliciet kiest voor server-side rendering.

Voor pagina's met dynamische gegevens zal de laadstatus van de pagina
pre-rendered worden.

#### Eerste render UX

In sommige gevallen kan de statische optimalisatie een ongewenste UX
veroorzaken waar de eerste render het ene laat zien, maar de tweede render
het andere. Dit gebeurt bijvoorbeeld bij het gebruik van `useSession()`.

In dit geval kan je `Page.suppressFirstRenderFlicker = true` instellen, en
Blitz zal de inhoud van de eerste render verbergen. Dit zal resulteren in
een kleine vertraging van de eerste render, maar zal de waargenomen UX
sterk verbeteren.

```tsx
const Page: BlitzPage = () => {
  return <div>{/* ... */}</div>
}

// highlight-start
Page.suppressFirstRenderFlicker = true
// highlight-end

export default Page
```

### Statische paginageneratie voor pagina's zonder authenticatie {#static-page-generation-for-unauthenticated-pages}

Voor pagina's die toegankelijk zijn voor iedereen zonder authenticatie,
raden we aan `getStaticProps` te gebruiken zodat de pagina, samen met zijn
data, 100% statisch gegenereerd wordt tijdens pre-rendering (zoals
Gatsby). Dan kan de hele statische pagina worden gecached op een CDN. Dit
is perfect voor publieke pagina's zoals blogposts.

Er zijn twee methodes voor gebruik met statische generatie, en je zal vaak
beide samen gebruiken.

1. `getStaticProps` - Om de **data** voor je pagina te laden. Zie de
   [`getStaticProps` documentatie](./get-static-props) voor meer details.
2. `getStaticPaths` - Om de mogelijke **paden** voor je pagina te laden.
   Zie de [`getStaticPaths` documentatie](./get-static-paths) voor meer
   details.

### Server-side rendering {#server-side-rendering}

> Ook wel "SSR" of "Dynamic Rendering" genoemd.

Als een pagina **server-side rendering** gebruikt, wordt de HTML van de
pagina bij **elke request** gegenereerd.

Om server-side rendering te gebruiken voor een pagina, moet je een `async`
functie met de naam `getServerSideProps` exporteren. Deze functie zal door
de server worden aangeroepen bij elke request.

Zie de [`getServerSideProps` documentatie](./get-server-side-props) voor
meer details.
