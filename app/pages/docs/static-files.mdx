---
title: Statische bestanden en afbeeldingen
sidebar_label: Statische bestanden en afbeeldingen
---

Blitz kan statische bestanden serven, zoals afbeeldingen, in de `public`
map in de root directory. Bestanden in `public` kunnen gerefereerd worden
in je code startend van de base URL (`/`).

Als je bijvoorbeeld een afbeelding toevoegt op
`public/mijn-afbeelding.png`, kan je deze gebruiken met de volgende code:

```jsx
function MyImage() {
  return <img src="/mijn-afbeelding.png" alt="Mijn afbeelding" />
}

export default MyImage
```

Deze map is ook handig voor `robots.txt`, Google Site Verification en
andere statische bestanden (waaronder `.html`-bestanden)!

> **Let op**: Verander de naam van de `public` map niet. De naam kan niet
> worden veranderd en dit is de enige map die gebruikt wordt voor
> statische bestanden.

> **Let op**: Zorg ervoor dat je geen statisch bestand hebt met dezelfde
> naam als een bestandsnaam in een `pages/` map. Dit zal resulteren in een
> fout.
>
> Lees meer: <http://err.sh/next.js/conflicting-public-file-page>
