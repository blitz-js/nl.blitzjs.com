Met Blitz kan je CSS-bestanden rechtstreeks in elk JavaScript-bestand importeren.

## Een globaal stylesheet toevoegen {#adding-a-global-stylesheet}

Om een stylesheet aan je applicatie toe te voegen, importeer je het CSS-bestand in `app/pages/_app.js`.

Neem bijvoorbeeld de volgende stylesheet met de naam `styles.css`:

```css
body {
  font-family: "SF Pro Text", "SF Pro Icons", "Helvetica Neue", "Helvetica", "Arial", sans-serif;
  padding: 20px 20px 60px;
  max-width: 680px;
  margin: 0 auto;
}
```

Importeer dan het `styles.css` bestand.

```jsx
import "../styles.css"

export default function App({Component, pageProps}) {
  return <Component {...pageProps} />
}
```

Deze stijlen (`styles.css`) zijn van toepassing op alle pagina's en componenten in je applicatie.
Vanwege het globale karakter van stylesheets, en om conflicten te vermijden, mag je ze **alleen importeren binnen [`_app.js`](./app-component)**.

Als je stylesheets op deze manier gebruikt, kunnen ze tijdens het ontwikkelen opnieuw geladen worden zodat je de status van de applicatie kan behouden.

In productie zullen alle CSS-bestanden automatisch worden samengevoegd tot één verkleind `.css` bestand.

## Toevoegen van CSS op componentniveau {#adding-component-level-css}

Blitz ondersteunt [CSS Modules](https://github.com/css-modules/css-modules) en maakt hiervoor gebruik van de `[name].module.css` bestandsnaamconventie.

CSS Modules geven CSS een lokale scope door automatisch een unieke klassenaam aan te maken. 
Hierdoor kan je dezelfde CSS klassenaam in verschillende bestanden gebruiken zonder je zorgen te hoeven maken over conflicten.

Dit gedrag maakt CSS Modules de ideale manier om CSS op componentniveau toe te voegen.
CSS Module-bestanden **kunnen overal in uw applicatie worden geïmporteerd**.


Neem bijvoorbeeld het herbruikbare `Button` component in de`components/` map:

Maak eerst `app/components/Button.module.css` met de volgende inhoud:

```css
/*
You do not need to worry about .error {} colliding with any other `.css` or
`.module.css` files!
*/
.error {
  color: white;
  background-color: red;
}
```

Maak vervolgens `app/components/Button.js` dat het bovenstaande CSS-bestand importeert en gebruikt:

```jsx
import styles from "./Button.module.css"

export function Button() {
  return (
    <button
      type="button"
      // Note how the "error" class is accessed as a property on the imported
      // `styles` object.
      className={styles.error}
    >
      Destroy
    </button>
  )
}
```

CSS Modules is een _optionele feature_ en wordt **enkel ingeschakeld voor bestanden met de `.module.css` extensie**. 
Gewone `<link>` stylesheets en globale CSS bestanden worden nog steeds ondersteund.

In productie zullen alle CSS Module-bestanden automatisch worden samengevoegd tot **vele verkleinde en code-split** `.css` bestanden.
Deze `.css` bestanden vertegenwoordigen hot execution paths in je applicatie, en zorgen ervoor dat de minimale hoeveelheid CSS wordt geladen om je applicatie weer te geven.

## Sass ondersteuning {#sass-support}

Met Blitz kan je zowel via de `.scss` als `.sass` extensies Sass importeren.
Je kan Sass op componentniveau gebruiken via CSS-modules en via een van de twee beschikbare extensies.

Voordat je de ingebouwde Sass-ondersteuning van Blitz kan gebruiken, moet je eerst [`sass`](https://github.com/sass/sass) installeren:

```bash
npm install sass
```

Sass-ondersteuning heeft dezelfde voordelen en beperkingen als de ingebouwde CSS-ondersteuning zoals hierboven beschreven.

### Aanpassen van Sass opties {#customizing-sass-options}

Als je de Sass-compiler wil aanpassen, kan je dat doen door gebruik te maken van `sassOptions` in `blitz.config.js`.

Om bijvoorbeeld `includePaths` toe te voegen:

```js
const path = require("path")

module.exports = {
  sassOptions: {
    includePaths: [path.join(__dirname, "styles")],
  },
}
```

## Less en Stylus ondersteuning {#less-and-stylus-support}

Om het importeren van `.less` of `.styl` bestanden te ondersteunen kan je de volgende plugins gebruiken:

- [@vercel/next-less](https://github.com/vercel/next-plugins/tree/master/packages/next-less)
- [@vercel/next-stylus](https://github.com/vercel/next-plugins/tree/master/packages/next-stylus)

Als je de less plugin gebruikt vergeet dan niet om ook een dependency van less toe te voegen, anders krijg je een foutmelding zoals:

```bash
Error: Cannot find module 'less'
```
